# AI Summaries (lite)

First Phase 4 AI feature. Per-thread structured summarization via Claude Haiku 4.5 with prompt caching on the system block (`cache_control: { type: "ephemeral" }`). **Tool-use output** locked at `{ tldr: string, ask?: string, decision?: string, deadline?: string }` per `decisions.md` — four chips in the UI, not a paragraph. **"Show me the prompt" trust modal** exposes the exact prompt + user payload + model + token usage of the call that produced any stored summary; `AISummary` rows record `promptVersion` and a frozen client-safe registry keeps old prompts indefinitely. **Prompt-injection defense** wraps email body in `<email>…</email>` tags (literal-tag-escape on the way in), with system-prompt instructions to treat tagged content as data; tested against a phishing fixture that must NOT yield "HACKED" in output. **Lazy generation** on thread open; **invalidate on new mail** via a one-line hook in the shared `writeDelta` writer; **rate-limited** at 30/min per user via an in-memory limiter (new `lib/ai/rate-limit.ts`). Long threads truncated to last 20 messages × 2 KB each. No streaming (small structured response). No back-fill. Schema adds one `AISummary` table; no provider-side changes. Builds the foundation that `ai-reply-drafts` and `ai-prioritization` reuse: `lib/ai/client.ts`, the rate limiter, and the prompt-injection guard.
