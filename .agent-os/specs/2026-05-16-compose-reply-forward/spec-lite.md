# Compose, Reply, Forward (lite)

First user-visible write surface. Adds four routes — `/inbox/compose/new`, `/inbox/[threadId]/(reply|reply-all|forward)` — sharing a single TipTap-based `<Composer />` client component. Reply / reply-all / forward pre-fill `to`, `subject` (with `Re:` / `Fwd:` de-double-prefix), `inReplyTo`, and the `References` chain from the parent message; forward injects the original body as quoted HTML. Account picker switches between connected `MailAccount`s (locked in reply modes). Attachments upload through a Server Action `FormData` and ride along to `IEmailProvider.sendMessage` / `reply` as `SendAttachment[]` — no long-lived blob storage. Server enforces 25 MB total + MIME deny list (executables, scripts). New `Draft` Prisma model with `@@unique([userId, threadId, mode])` enables resumable autosave debounced 2 s after typing. Provider-agnostic; no `if (provider === ...)` in `app/inbox/compose/**`. Out of scope: AI drafts, recipient autocomplete, scheduled send, signature editor, IndexedDB offline queue.
