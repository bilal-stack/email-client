# AI Reply Drafts (lite)

Second Phase 4 AI feature. AI-generated reply drafts in **three tone variants** (`terse` / `friendly` / `detailed`) returned in **one streamed Anthropic call** via tool-use — locked in `decisions.md`. **Sonnet 4.6** (quality matters for outgoing prose). **Prompt-cached system block.** **Streaming** mandatory per CLAUDE.md "stream anything > ~1s" — `messages.stream` + RSC `createStreamableValue` exposes three streamable strings the panel populates progressively as the model emits completed segments of the tool's JSON. Tone-matched from the user's last 5 sent messages on the same account (empty history → neutral professional tone). **Reuses** `lib/ai/client.ts`, `lib/ai/rate-limit.ts`, `lib/ai/prompt-injection-guard.ts` from `ai-summaries`. **Prompt-injection defense** carries over: email bodies wrapped in `<email>…</email>` with ZWJ escapes; sent samples in `<sent-samples>…</sent-samples>`. **Prompt size capped** at last-20-messages × 2 KB + 5 sent samples × 1 KB. **No new schema** — accepted variants flow into the existing `Draft` row through the composer's autosave path; an `AIDraft` table is explicitly deferred. The UI is a three-tab panel mounted in the reply / reply-all / forward composer, with an overwrite-confirm if the user already has unsent edits.
