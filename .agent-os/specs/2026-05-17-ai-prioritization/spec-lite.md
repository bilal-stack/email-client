# AI Prioritization (lite)

Third Phase 4 AI feature. Runs **Haiku 4.5** on every new message via Inngest event-trigger (`inbox/message.created` emitted from the shared `writeDelta` writer post-commit), per-user concurrency cap 2. Tool-use schema locked in `decisions.md`: `{ priority: 1-5, reason: string ≤6 words, suggestedActions: ("reply"|"archive"|"snooze"|"delegate")[], riskFlag: "phish"|"promo"|"ok" }`. **The differentiator**: the inbox row shows the `reason` as a chip — *why* this matters, not just a number — and `riskFlag !== "ok"` adds a colored trust badge. **The inbox defaults to AI-priority sort**; a header toggle restores chronological. Re-prioritization on every new message: a thread can rise when new context arrives. Reuses `lib/ai/client.ts`, `lib/ai/rate-limit.ts`, `lib/ai/prompt-injection-guard.ts` from earlier AI specs. **Belt-and-suspenders sanitization** strips `<`/`>`/`http(s)://` from the `reason` field and caps at 6 words — defense in depth against prompt-injection content surfacing in the chip. **`snooze` / `delegate` `suggestedActions` are tooltip-only** in this spec; their action paths are deferred. New Prisma table `PriorityScore` keyed `(messageId)` unique. Adds `?sort=time|priority` URL param to the inbox; default `priority`. Threads without scores yet show a placeholder chip until the SSE `priority-updated` event lands.
