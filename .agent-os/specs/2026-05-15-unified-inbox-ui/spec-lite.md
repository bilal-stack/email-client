# Unified Inbox UI (lite)

First user-visible mail surface. `/inbox` renders the signed-in user's `Thread` rows from the DB (never calls the provider during a render), with an account-switcher filter persisted in the URL search param `?account=`. `/inbox/[threadId]` lists messages oldest → newest with HTML bodies rendered inside a sandboxed iframe (DOMPurify + tracker strip + strict CSP per the `email-html-sanitize` skill). Opening a thread auto-fires a Server Action that calls the adapter's `markRead` and updates local `Message.isUnread` rows. Real-time updates flow via SSE at `/api/inbox/events`, fed by an in-process Node `EventEmitter` the Inngest sync function writes to after commit. Provider-agnostic — no `if (provider === ...)` branches. Mobile: list-only <768 px, split-pane ≥768 px. Out of scope: compose, search, labels, archive, AI surfaces.
