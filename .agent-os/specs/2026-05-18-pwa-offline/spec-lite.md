# PWA Offline (lite)

First Phase 5 spec. Installable PWA + offline-capable inbox via **Serwist** (the maintained Workbox successor; `@serwist/next` + `serwist` + `idb`). Custom service worker at `app/sw.ts` declares four runtime caches: app-shell (CacheFirst on inbox routes), static-assets (CacheFirst on `/_next/static`), inbox-data (StaleWhileRevalidate on GET mirror routes for the list / thread Server Actions), thread-bodies (CacheFirst, 7-day TTL, 50 entries). Network-only for `/api/auth/**` — OAuth callbacks bypass the SW entirely. Network-first elsewhere with `app/offline/page.tsx` as the terminal fallback. Web App Manifest via Next.js's `app/manifest.ts` (display standalone, start_url `/inbox`); icons (192 / 512 / 512-maskable) committed as flat assets. **Offline compose**: the composer's autosave path detects `navigator.onLine === false` and queues drafts to IndexedDB via `lib/offline/draft-queue.ts`; `lib/offline/draft-replay.ts` listens to the `online` event and pushes the queue through the existing `upsertDraft` Server Action. **No offline send**, **no offline AI** (the summary banner degrades with "unavailable offline"; AI draft button disabled; cached priority chips still render). Service worker disabled in dev (`npm run dev`); production-only registration. Lighthouse PWA target: ≥ 90. SW skipWaiting + clients.claim so new builds take over on next page load.
