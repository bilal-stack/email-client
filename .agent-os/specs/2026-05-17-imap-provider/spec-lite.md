# IMAP Provider (lite)

Third concrete `IEmailProvider` adapter using `imapflow` + `nodemailer` for Yahoo / AOL / generic IMAP mailboxes. Fleshes out the Auth.js Credentials `authorize` stub from `foundation` into a real path that validates the IMAP login, runs the SSRF host-guard (RFC1918 / loopback / link-local rejected in production), encrypts the app-password, and upserts a `MailAccount` with `provider: "imap"`. `lib/providers/auth.ts` `MailboxSecret` becomes a discriminated union (`kind: "oauth" | "imap"`); decoding a legacy row without `kind` treats it as OAuth (no DB migration). Sync uses **UID-range polling, not IDLE** — same 60-second Inngest cron as gmail-sync / graph-sync, anchored on a `UIDVALIDITY:HIGHEST_UID` cursor; a `UIDVALIDITY` flip maps to `AuthError` ("Mailbox state reset — reconnect required") in parallel with Gmail's stale-historyId and Graph's stale-deltaLink. System-label round-trip via folder-move + flag operations (`INBOX`/`SENT`/`DRAFT`/`TRASH`/`UNREAD`/`STARRED`); user labels are a silent no-op at the adapter boundary (IMAP has no equivalent — local DB still reflects them, just no server round-trip). Threading reconstructed from `In-Reply-To`/`References` against existing `Message.providerMessageId`s; mint a new thread if no match. **No schema changes** — the Phase 2 tables are already provider-neutral. **No UI changes** — the inbox / composer / search / labels surfaces consume `IEmailProvider` and must light up unchanged.
